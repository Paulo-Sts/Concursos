# Pensamento Computacional

## 1. Definição e Objetivo

#### 1.1 Definição
- Abordagem que utiliza conceitos da Ciência da Computação para resolver problemas de forma estruturada e lógica.
- Base para o desenvolvimento de software e automação de tarefas.

#### 1.2 Objetivo
- Estruturar soluções para problemas de forma lógica e eficiente.
- Facilitar a compreensão de como o computador processa informações e executa comandos.

## 2. Pilares do Pensamento Computacional (4 Dimensões)

#### 2.1 Decomposição
- Dividir problemas complexos em partes menores e mais gerenciáveis.
- Abordagem conhecida como "dividir para conquistar".
- Exemplo: Divisão de grandes projetos (como a transposição do Rio São Francisco) em etapas menores.

#### 2.2 Reconhecimento de Padrões
- Identificar padrões, tendências ou regularidades em dados para simplificar a solução.
- Permite a criação de estruturas de repetição (loops) em algoritmos.
- Exemplo: Algoritmo para calcular a média de notas de vários alunos, repetindo o processo para cada um.

#### 2.3 Abstração
- Focar nas informações relevantes e ignorar detalhes irrelevantes.
- Simplificar o problema, reduzindo sua complexidade.
- Pensar de forma genérica, considerando apenas os aspectos essenciais.
- Exemplo: Desenvolver um algoritmo para fazer um bolo, focando nos passos principais e ignorando variações pessoais da receita.

#### 2.4 Algoritmos
- Desenvolver uma sequência de passos ou regras claras e ordenadas para resolver o problema.
- Representação do processo de solução, podendo ser descrita em linguagem natural (ex: português) ou em linguagem de programação.
- Exemplo: Algoritmo em português para "Fazer Chá Rápico", com passos sequenciais e comandos como "aguarde 180 segundos".

## 3. Características dos Algoritmos

#### 3.1 Sequência Lógica
- Passos ordenados e claros que devem ser seguidos para atingir o resultado.

#### 3.2 Comentários
- Texto explicativo inserido no código (marcado por "//" em muitos contextos) que é ignorado durante a execução.
- Serve para documentar e facilitar o entendimento do algoritmo.

#### 3.3 Tradução para Linguagens de Programação
- Algoritmos em linguagem natural podem ser traduzidos para sintaxes de linguagens de programação (geralmente baseadas em inglês, ex: "print" no lugar de "escreva").

## 4. Aplicações do Pensamento Computacional

#### 4.1 Não Restrito à Programação
- Aplicável em diversas áreas e no dia a dia para estruturação de soluções.
- Muitas pessoas utilizam esse raciocínio intuitivamente em situações cotidianas.

#### 4.2 Base para a Programação
- Habilidade fundamental para quem desenvolve software.
- Permite a automação de processos por meio de linguagens de programação.

#### 4.3 Relação com Outros Conhecimentos
- Está ligado ao estudo da Lógica.
- Envolve compreensão de sistemas de numeração, especialmente o sistema binário (0 e 1) usado pelos computadores.

## 5. Pontos Relevantes para Concursos
1. Os quatro pilares são: Decomposição, Reconhecimento de Padrões, Abstração e Algoritmos.
2. A automação é representada pelo algoritmo (mecanização da solução).
3. Estratégias pedagógicas comuns incluem: Algoritmo, Abstração, Descrição, Reflexão e Depuração.
4. O pensamento computacional é uma habilidade transversal, útil mesmo para quem não é programador.
