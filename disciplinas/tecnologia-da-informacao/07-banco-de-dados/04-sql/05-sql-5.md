# Anotações – Banco de Dados – Linguagem SQL V (ANY, ALL, EXISTS, UNION, INTERSECT, EXCEPT)

## 1. Operadores de Comparação com Subconsultas: ANY e ALL

### 1.1 ANY

- **`ANY`** compara um valor com **qualquer** um dos valores retornados pela subconsulta.
- Retorna `TRUE` se a condição for verdadeira para **pelo menos um** valor da subconsulta.

**Exemplo:**

```sql
SELECT * FROM Empregado
WHERE idade > ANY (SELECT idade FROM Empregado WHERE depto = 'ENG');
```

**Tabela EMPREGADO:**

| ID_Emp | Nome | Depto | Idade |
| --- | --- | --- | --- |
| 1 | Joao | TI | 25 |
| 2 | Maria | FIN | 38 |
| 3 | Marcos | FIN | 44 |
| 4 | Renata | JUR | 27 |
| 5 | Joana | JUR | 22 |
| 6 | Juliana | TI | 24 |
| 7 | Marcelo | TI | 28 |
| 8 | Artur | ENG | 35 |
| 9 | Gustavo | ENG | 37 |
| 10 | Marta | ENG | 39 |

**Subconsulta (idades do ENG):** `35, 37, 39`

- **ANY** → `idade > 35` (maior que o **menor** valor da subconsulta).
- Resultado: empregados com idade **maior que 35** – Maria (38), Marcos (44), Gustavo (37), Marta (39).

> ### 1.1.1 Regra prática
> - `> ANY` → maior que o **menor** valor.
> - `< ANY` → menor que o **maior** valor.
> - `= ANY` → equivalente a `IN`.

### 1.2 ALL

- **`ALL`** compara um valor com **todos** os valores retornados pela subconsulta.
- Retorna `TRUE` se a condição for verdadeira para **todos** os valores da subconsulta.

```sql
SELECT * FROM Empregado
WHERE idade > ALL (SELECT idade FROM Empregado WHERE depto = 'ENG');
```

**Subconsulta (idades do ENG):** `35, 37, 39`

- **ALL** → `idade > 39` (maior que o **maior** valor da subconsulta).
- Resultado: apenas Marcos (44).

> ### 1.2.1 Regra prática
> - `> ALL` → maior que o **maior** valor.
> - `< ALL` → menor que o **menor** valor.
> - `= ALL` → raramente usado; retorna verdadeiro se o valor for igual a **todos** os valores (equivale a `IN` com um único valor).

### 1.3 Comparação ANY vs. ALL

| SUBQUERY (ENG idades) = [35, 37, 39] | ANY | ALL |
| --- | --- | --- |
| `idade >` | > 35 | > 39 |
| `idade <` | < 39 | < 35 |
| `idade =` | equivale a `IN` (35,37,39) | difícil de ser verdadeiro |

## 2. EXISTS – Verificação de Existência

- **`EXISTS`** verifica se a subconsulta retorna **pelo menos uma linha**.
- Retorna `TRUE` se a subconsulta tiver pelo menos um resultado.

### 2.1 Exemplo

```sql
SELECT * FROM Empregado E
WHERE NOT EXISTS (SELECT * FROM Dependente D WHERE E.CPF = D.CPF);
```

**Tabela EMPREGADO:**

| CPF | Nome | Depto |
| --- | --- | --- |
| 111 | Joao | TI |
| 222 | Maria | FIN |
| 333 | Marcos | FIN |
| 444 | Renata | JUR |
| 555 | Joana | JUR |

**Tabela DEPENDENTE:**

| CPF | Nome |
| --- | --- |
| 111 | Joao Jr. |
| 111 | Joao Neto |
| 333 | Marcos Jr. |
| 444 | Renato |

- `NOT EXISTS` retorna apenas os empregados **que não têm dependentes**: Maria (222) e Joana (555).

> ### 2.1.1 `EXISTS` vs. `IN`
> - `EXISTS` é **mais eficiente** para subconsultas que retornam muitas colunas.
> - `IN` é usado para comparar um valor com uma lista de valores (retorna uma coluna).
> - `EXISTS` é frequentemente usado com **subconsultas correlacionadas**.

## 3. UNION e UNION ALL – Combinação de Resultados

### 3.1 UNION

- Combina os resultados de **duas ou mais consultas**.
- **Remove duplicatas** automaticamente.
- Exige que as consultas tenham o **mesmo número de colunas** e tipos compatíveis.

```sql
SELECT Nome, Depto FROM Servidor
UNION
SELECT Nome, Depto FROM Terceirizado;
```

**Tabela SERVIDOR:**

| Nome | Depto |
| --- | --- |
| Joao | TI |
| Maria | FIN |
| Marcos | FIN |
| Renata | JUR |
| Joana | JUR |

**Tabela TERCEIRIZADO:**

| Nome | Depto |
| --- | --- |
| Vinicius | TI |
| Maria | FIN |
| Miguel | FIN |
| Andre | JUR |

**Resultado (UNION):**

| Nome | Depto |
| --- | --- |
| Joao | TI |
| Maria | FIN |
| Marcos | FIN |
| Renata | JUR |
| Joana | JUR |
| Vinicius | TI |
| Miguel | FIN |
| Andre | JUR |

- Maria aparece apenas uma vez (duplicata removida).

### 3.2 UNION ALL

- Combina os resultados **sem remover duplicatas**.
- É mais rápido que `UNION` (não precisa verificar duplicatas).

```sql
SELECT Nome, Depto FROM Servidor
UNION ALL
SELECT Nome, Depto FROM Terceirizado;
```

**Resultado (UNION ALL) – Maria aparece duas vezes:**

| Nome | Depto |
| --- | --- |
| Joao | TI |
| Maria | FIN |
| Marcos | FIN |
| Renata | JUR |
| Joana | JUR |
| Maria | FIN |
| Vinicius | TI |
| Miguel | FIN |
| Andre | JUR |

> ### 3.2.1 Quando usar cada um
> - `UNION` – quando se deseja eliminar duplicatas.
> - `UNION ALL` – quando se deseja todas as linhas (incluindo duplicatas) e melhor performance.

## 4. INTERSECT – Interseção de Conjuntos

- Retorna os registros que **estão presentes em ambas as consultas**.
- Equivalente à **interseção** de conjuntos.

```sql
SELECT Nome FROM Conta
INTERSECT
SELECT Nome FROM Emprestimo;
```

**Tabela CONTA (clientes com conta):**

| Nome |
| --- |
| Joao |
| Maria |
| Rafael |
| Julio |
| Renata |

**Tabela EMPRESTIMO (clientes com empréstimo):**

| Nome |
| --- |
| Maria |
| Rafael |
| Julio |
| Antonio |

**Resultado (INTERSECT):**

| Nome |
| --- |
| Maria |
| Rafael |
| Julio |

> ### 4.1 Observação
> - `INTERSECT` só retorna os nomes que aparecem **em ambas as tabelas**.

## 5. EXCEPT – Diferença de Conjuntos

- Retorna os registros que estão na **primeira consulta** mas **não estão na segunda**.
- Equivalente à **diferença** de conjuntos.

```sql
SELECT Nome FROM Conta
EXCEPT
SELECT Nome FROM Emprestimo;
```

**Resultado (EXCEPT):**

| Nome |
| --- |
| Joao |
| Renata |

> ### 5.1 Ordem importa
> - `A EXCEPT B` → elementos que estão em **A** mas **não** em **B**.
> - `B EXCEPT A` → elementos que estão em **B** mas **não** em **A** (Antonio, no exemplo).

## 6. Síntese Comparativa – Operadores de Conjunto

| OPERADOR | FUNÇÃO | REMOVE DUPLICATAS? | EQUIVALÊNCIA MATEMÁTICA |
| --- | --- | --- | --- |
| `UNION` | Combina dois conjuntos. | **Sim** | União |
| `UNION ALL` | Combina dois conjuntos (mantém duplicatas). | **Não** | União (sem remoção de duplicatas) |
| `INTERSECT` | Retorna apenas elementos comuns. | **Sim** | Interseção |
| `EXCEPT` | Retorna elementos do primeiro que não estão no segundo. | **Sim** | Diferença (A – B) |

> ### 6.1 Observação sobre `EXCEPT` nos SGBDs
> - No **SQL Server**, usa-se `EXCEPT`.
> - No **Oracle**, usa-se `MINUS` para a mesma operação.

## 7. Regras para Union, Intersect e Except

| REGRA | DESCRIÇÃO |
| --- | --- |
| **Mesmo número de colunas** | Todas as consultas devem ter o mesmo número de colunas. |
| **Tipos compatíveis** | Os tipos de dados das colunas correspondentes devem ser compatíveis. |
| **Ordem dos campos** | A ordem das colunas nas consultas deve corresponder. |
| **Remoção de duplicatas** | `UNION`, `INTERSECT` e `EXCEPT` removem duplicatas por padrão. |
