# Anotações – Banco de Dados – Linguagem SQL II (SELECT, WHERE, Operadores)

## 1. SELECT – Selecionando Colunas e Filtrando Dados

### 1.1 Selecionar colunas específicas

```sql
SELECT Nome, Preco FROM Produto;
```

- Retorna apenas as colunas `Nome` e `Preco` da tabela `Produto`.

### 1.2 Filtrar com `WHERE`

```sql
SELECT Nome, Preco FROM Produto WHERE Categoria = 'Bebida';
```

- Retorna apenas os produtos cuja categoria seja "Bebida".

### 1.3 `SELECT *`

```sql
SELECT * FROM Produto;
```

- Retorna **todas as colunas** da tabela.

### 1.4 Filtrar por condição numérica

```sql
SELECT * FROM Produto WHERE Preco > 10;
```

- Retorna produtos com preço **maior que 10** (exclui exatamente 10).

## 2. Ordenação com `ORDER BY`

```sql
SELECT * FROM Produto ORDER BY Nome;
```

- Ordena os resultados em **ordem crescente (ASC)** por padrão.
- Para ordenação decrescente: `ORDER BY Nome DESC`.

**Resultado (exemplo com produtos):** Açúcar, Café, Refrigerante, Sabão em Pó, Vinho.

## 3. Eliminando Duplicidades com `DISTINCT`

```sql
SELECT DISTINCT Categoria FROM Produto;
```

- Remove linhas duplicadas no resultado.
- Útil para listar valores únicos de uma coluna.

> ### 3.1 Exemplo
> - Se houver 5 produtos com 3 categorias (Mercearia, Bebida, Limpeza), o `SELECT DISTINCT Categoria` retornará apenas 3 linhas.

## 4. Operador `LIKE` – Padrões de Strings

| CORINGA | SIGNIFICADO | EXEMPLO |
| --- | --- | --- |
| `%` | Qualquer sequência de caracteres (zero ou mais) | `LIKE 'A%'` → começa com A |
| `_` | Exatamente um caractere | `LIKE '_a%'` → segunda letra é 'a' |

```sql
SELECT * FROM Produto WHERE Nome LIKE '%a%';
```

- Retorna todos os produtos cujo nome contenha a letra "a" em qualquer posição.

> ### 4.1 Combinações comuns
> - `'A%'` – começa com "A".
> - `'%s'` – termina com "s".
> - `'%or%'` – contém "or".
> - `'_a%'` – segunda letra é "a".

## 5. Operador `IN` – Comparação com um Conjunto

```sql
SELECT Nome, Preco FROM Produto WHERE Categoria IN ('Bebida', 'Limpeza');
```

- Retorna produtos cuja categoria esteja na lista fornecida.
- Equivalente a `Categoria = 'Bebida' OR Categoria = 'Limpeza'`, mas mais legível e eficiente.

### 5.1 `NOT IN`

```sql
SELECT * FROM Produto WHERE Categoria NOT IN ('Bebida', 'Limpeza');
```

- Retorna produtos cuja categoria **não** esteja na lista.

## 6. Subconsultas (Subqueries)

- Uma subconsulta é uma consulta SQL aninhada dentro de outra consulta.
- A subconsulta interna é executada primeiro, e seu resultado é usado pela consulta externa.

### 6.1 Exemplo com `IN` e subconsulta

```sql
SELECT Nome, Preco
FROM Produto
WHERE Categoria_ID IN (
    SELECT ID FROM Categoria WHERE Nome IN ('Bebida', 'Limpeza')
);
```

**Passos:**
1. Subconsulta: `SELECT ID FROM Categoria WHERE Nome IN ('Bebida', 'Limpeza')` → retorna, por exemplo, `(30, 20)`.
2. Consulta externa: `SELECT Nome, Preco FROM Produto WHERE Categoria_ID IN (30, 20)`.

> ### 6.1.1 Vantagem da subconsulta
> - Permite consultas mais dinâmicas: a lista de IDs não precisa ser conhecida previamente.
> - Útil quando os critérios de filtro estão em outra tabela.

## 7. Operadores Lógicos – `AND`, `OR`, `NOT`

| OPERADOR | COMPORTAMENTO | EXEMPLO |
| --- | --- | --- |
| `AND` | Ambas as condições devem ser verdadeiras. | `WHERE Preco > 10 AND Categoria = 'Bebida'` |
| `OR` | Pelo menos uma condição deve ser verdadeira. | `WHERE Categoria = 'Bebida' OR Categoria = 'Limpeza'` |
| `NOT` | Inverte a condição. | `WHERE NOT Categoria = 'Bebida'` |

## 8. Operadores Especiais – Resumo

| OPERADOR | DESCRIÇÃO | EXEMPLO |
| --- | --- | --- |
| `=` | Igual | `WHERE codigo = 10` |
| `<>` / `!=` | Diferente | `WHERE codigo <> 2` |
| `<`, `>`, `<=`, `>=` | Comparação | `WHERE preco > 500` |
| `BETWEEN` | Intervalo (inclusivo) | `WHERE preco BETWEEN 10 AND 100` |
| `LIKE` | Padrão de string | `WHERE nome LIKE 'A%'` |
| `IN` | Pertence a um conjunto | `WHERE id IN (2, 5, 15)` |
| `IS NULL` / `IS NOT NULL` | Verifica valor nulo | `WHERE preco IS NULL` |

> ### 8.1 Cuidado com `NULL` (pegadinha!)
> - **`NULL`** representa **ausência de valor**.
> - Não se pode usar `= NULL`. A comparação correta é `IS NULL` ou `IS NOT NULL`.
> - Exemplo: `WHERE preco = NULL` **está errado**; o correto é `WHERE preco IS NULL`.

## 9. Tabela-verdade para `AND` e `OR`

| A | B | A AND B | A OR B |
| --- | --- | --- | --- |
| V | V | V | V |
| V | F | F | V |
| F | V | F | V |
| F | F | F | F |

> ### 9.1 Aplicação no SQL
> - `AND` – mais restritivo (exige ambas as condições).
> - `OR` – menos restritivo (basta uma condição verdadeira).

## 10. Boas Práticas e Dicas para Provas

| DICA | EXPLICAÇÃO |
| --- | --- |
| **Teste comandos com `NULL`** | Lembre-se: `IS NULL`, nunca `= NULL`. |
| **Use `DISTINCT` com cuidado** | Pode alterar o resultado em `JOIN`, mas em tabelas simples é seguro. |
| **Prefira `IN` a múltiplos `OR`** | Mais legível e com melhor desempenho. |
| **Ordene com `ORDER BY`** | Padrão é `ASC`; use `DESC` para decrescente. |
| **Entenda a ordem de execução** | Subconsulta mais interna → consulta externa. |
| **`LIKE` é sensível a maiúsculas?** | Depende do SGBD; na teoria, considere que `LIKE` é sensível a maiúsculas/minúsculas. |
