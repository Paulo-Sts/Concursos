# Anotações – PLN – Vetorização

## 1. Etapas do Processamento de Linguagem Natural

1. Coleta dos dados
2. Pré-processamento
3. **Vetorização:** transformação das palavras em números (vetores) – entrada para a rede neural
4. Modelagem

## 2. Bag-of-Words (BoW)

- **Técnica de vetorização** que conta quantas vezes cada palavra aparece em um documento e cria um vetor com essas contagens.

### 2.1 Funcionamento

1. Cria-se um vocabulário com todas as palavras únicas do corpus.
2. Cada documento é representado por um vetor onde cada posição corresponde a uma palavra do vocabulário.
3. O valor em cada posição é a **frequência** da palavra no documento.

### 2.2 Características

| VANTAGENS | DESVANTAGENS |
| --- | --- |
| Fácil de implementar | Ignora a **ordem** das palavras |
| Bom para classificações simples | Ignora o **contexto** |
| Rápido | Vetores **enormes e esparsos** (muitos zeros) |

> ### 2.2.1 Exemplo
> - Documento 1: "Eu amo o Gran" → após pré-processamento: ["am", "Gran"]
> - Vetor (vocabulário: am, Gran, pref, leg): `[1, 1, 0, 0]`

## 3. TF-IDF

- **TF-IDF (Term Frequency – Inverse Document Frequency):** técnica que pondera cada termo considerando sua frequência no documento (TF) e sua raridade no conjunto de documentos (IDF).

### 3.1 Fórmulas

```
TF(t, d) = número de vezes que t aparece em d

IDF(t) = log(N / df_t)
onde:
N = número total de documentos
df_t = número de documentos que contêm o termo t

TF-IDF(t, d) = TF(t, d) × IDF(t)
```

### 3.2 Características

| VANTAGENS | DESVANTAGENS |
| --- | --- |
| Dá mais peso a palavras que distinguem documentos | Ainda ignora ordem e semântica |
| Reduz peso de palavras muito comuns | Vetores ainda grandes e esparsos |
| Mais discriminativo que BoW | |

> ### 3.2.1 Interpretação
> - Se uma palavra aparece **muitas vezes em um documento** e **poucas vezes nos outros**, seu TF-IDF será **alto** → palavra importante para aquele documento.

## 4. Word Embeddings

- Técnica que transforma palavras em **vetores densos** (de baixa dimensionalidade).
- Palavras com sentidos semelhantes têm vetores **próximos** (semanticamente similares → geometricamente próximos).

### 4.1 Vantagens sobre BoW e TF-IDF

| CARACTERÍSTICA | BoW / TF-IDF | Word Embeddings |
| --- | --- | --- |
| Dimensionalidade | Alta (esparsa) | Baixa (densa) |
| Captura semântica? | Não | **Sim** |
| Relações entre palavras | Não captura | Captura (ex.: rei – homem + mulher ≈ rainha) |
| Contexto | Ignora | Captura (depende do modelo) |

## 5. Principais Modelos de Word Embeddings

### 5.1 Word2Vec (Google)

- **Framework** que engloba dois modelos:
  - **CBOW (Continuous Bag-of-Words):** prevê a palavra central a partir do contexto.
  - **Skip-gram:** prevê o contexto a partir da palavra central.
- Utiliza uma **rede neural rasa** (não profunda).
- Captura relações semânticas.

| MODELO | FUNCIONAMENTO | MELHOR PARA |
| --- | --- | --- |
| **CBOW** | Contexto → palavra central | Palavras frequentes, treino mais rápido |
| **Skip-gram** | Palavra central → contexto | Palavras raras, captura relações mais profundas |

> ### 5.1.1 Exemplo (analogia)
> - `rei – homem + mulher ≈ rainha`
> - `Paris – França ≈ Roma – Itália`

### 5.2 GloVe (Global Vectors)

- Combina:
  - **Estatísticas globais de cocorrência** (como ocorrência de "João Pessoa" juntos).
  - **Representação vetorial densa** (como Word2Vec).
- Utiliza **fatoração de matriz** a partir de estatísticas do corpus inteiro.

| VANTAGENS | DESVANTAGENS |
| --- | --- |
| Captura relações globais | Não contextual (mesma palavra → mesmo vetor) |
| Estável e eficiente | Não diferencia sentidos de palavras polissêmicas |
| Excelente para grandes corpora | |

### 5.3 Limitação de todos os embeddings estáticos

- Palavras com múltiplos sentidos (ex.: "manga" – fruta ou blusa) têm **o mesmo vetor**.
- Não diferenciam contexto.

## 6. Comparação entre Técnicas

| MÉTODO | TIPO | É EMBEDDING? | COMO FUNCIONA | PONTOS FORTES | LIMITAÇÕES |
| --- | --- | --- | --- | --- | --- |
| **BoW** | Contagem | Não | Conta palavras; ignora ordem/contexto | Simples, rápido | Vetores enormes/esparsos; sem semântica |
| **TF-IDF** | Contagem ponderada | Não | Pondera por importância no corpus | Diferencia termos relevantes | Sem semântica; vetores esparsos |
| **CBOW** | Neural | Sim (estático) | Prediz palavra-alvo a partir do contexto | Rápido, eficiente | Mesma palavra → mesmo vetor |
| **Skip-gram** | Neural | Sim (estático) | Prediz contexto a partir da palavra central | Captura relações semânticas profundas | Treino mais lento; vetor fixo |
| **Word2Vec** | Framework | Sim (estático) | Embeddings densos; captura analogias | Captura semântica, rápido | Não contextual |
| **GloVe** | Estatístico | Sim (estático) | Fatoração de matriz de cocorrência | Estável, robusto, excelente para grandes corpora | Não contextual |
