## 1. Hierarquia de Views (Visões)

### 1.1 Conceitos Fundamentais

- **`View`:** Blocos básicos de construção da interface visual. Qualquer elemento visível na tela é uma *View* (botão, texto, campo de entrada).
- **`Superview` e `Subview`:** As *Views* são organizadas hierarquicamente. Uma *View* principal (*Superview*) pode **hospedar** outras *Views* (*Subviews*).
- **Classe Raiz:** A classe base para todos os elementos visuais é a **`UIView`**.
- **`UIControl`:** Subclasse de `UIView` que define comportamentos específicos para elementos interativos como **botões** e **interruptores**.

> ### 1.1.1 Capacidades das Views
- Responder a toques.
- Desenhar conteúdo customizado (via *Core Graphics* ou *UIKit*).
- Suportar interações de **arrastar e soltar** (*drag and drop*).
- Responder a mudanças de foco.
- Realizar animações de tamanho, posição e aparência.

## 2. View Controllers (Controladores de Visão)

### 2.1 Definição e Responsabilidades

- **Função:** Orquestrar e gerenciar uma **cena** (conjunto de *Views*). Cada tela do aplicativo é gerenciada por um *View Controller*.
- **Estrutura:** Cada *app* possui ao menos um *View Controller* (que preenche a tela principal). Um *View Controller* controla uma **raiz** (*root view*) que contém outras *Subviews*.
- **Responsabilidades:**
    - Preencher *Views* com dados e atualizar mudanças.
    - Avisar mudanças de dados para os *Model Objects*.
    - Ajustar tamanho, posição e visibilidade das *Views*.
    - Facilitar a **transição** entre telas.

### 2.2 Navegação

- **`Navigation Controller`:** Componente responsável por gerenciar a **navegação hierárquica** entre telas (ex: "idas e voltas" entre uma listagem e um formulário).

### 2.3 Ciclo de Vida do View Controller

| MÉTODO | MOMENTO DE INVOCAÇÃO | USO PRINCIPAL |
| :--- | :--- | :--- |
| **`viewDidLoad()`** | Chamado quando a *View* do Controller é **criada** pela primeira vez. | **Configurações iniciais** (únicas). |
| **`viewWillAppear()`** | **Imediatamente antes** da *View* ser adicionada à hierarquia de visualização. | Acionar operações que precisam ocorrer **antes** da tela aparecer (ex: atualizar dados). Pode ser chamado várias vezes. |
| **`viewDidAppear()`** | **Logo após** a *View* ser exibida. | Acionar algo assim que a tela estiver visível (ex: iniciar animação). |
| **`viewWillDisappear()`** | **Imediatamente antes** da *View* ser removida da hierarquia. | Salvar estado, interromper tarefas. |
| **`viewDidDisappear()`** | **Logo após** a *View* ser removida. | Limpeza de recursos, parar processos. |

- **Resposta:** Deve-se utilizar o **`Navigation Controller`**.

### 3.4 Protocolo Equatable (AOCP/2019)

- **Função:** Protocolo que determina **quando dois objetos são iguais**.
- **Nome:** **`Equatable`**.

### 3.5 Análise de Código Swift (CESGRANRIO/2021)

- **Código:**
    ```swift
    var num = 0
    var frase = ""
    for i in 1...3 {
        num = num + 1 + i * 2
        switch num {
            case 2...6: texto = "a "
            case 7...9: texto = "casa "
            case 14...16: texto = "eh "
            default: texto = ""
        }
        frase = frase + texto
    }
    print(frase)
    ```
- **Iterações:**
    1.  `i=1`: `num = 0 + 1 + 2 = 2`. `switch 2` -> `case 2...6`: `texto = "a "`. `frase = "a "`.
    2.  `i=2`: `num = 2 + 1 + 4 = 7`. `switch 7` -> `case 7...9`: `texto = "casa "`. `frase = "a casa "`.
    3.  `i=3`: `num = 7 + 1 + 6 = 14`. `switch 14` -> `case 14...16`: `texto = "eh "`. `frase = "a casa eh "`.
- **Resultado Final:** `"a casa eh "`.
- **Gabarito:** Alternativa **B**.
