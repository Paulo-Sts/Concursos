# Anotações – Banco de Dados – Linguagem SQL III (Agregação, GROUP BY, HAVING, INSERT, DELETE, UPDATE)

## 1. Funções de Agregação

### 1.1 Visão geral

| FUNÇÃO | DESCRIÇÃO | EXEMPLO |
| --- | --- | --- |
| `COUNT(*)` | Conta o número total de registros. | `SELECT COUNT(*) FROM Produto;` |
| `AVG(coluna)` | Calcula a média dos valores da coluna. | `SELECT AVG(Preco) FROM Produto;` |
| `SUM(coluna)` | Soma os valores da coluna. | `SELECT SUM(Preco) FROM Produto;` |
| `MAX(coluna)` | Retorna o maior valor da coluna. | `SELECT MAX(Preco) FROM Produto;` |
| `MIN(coluna)` | Retorna o menor valor da coluna. | `SELECT MIN(Preco) FROM Produto;` |

> ### 1.1.1 Alias (apelido)
> - Usa-se `AS` para dar um nome à coluna resultante.
> - Exemplo: `SELECT COUNT(*) AS Total FROM Produto;`

## 2. `GROUP BY` – Agrupamento de Dados

- Agrupa os registros com base em valores de uma ou mais colunas.
- As funções de agregação são aplicadas **dentro de cada grupo**.

### 2.1 Exemplo com `GROUP BY`

**Tabela Produto:**

| Nome | Categoria | Preco |
| --- | --- | --- |
| Café | Mercearia | 8.99 |
| Açúcar | Mercearia | 10.20 |
| Sabão em Pó | Limpeza | 9.90 |
| Vinho | Bebida | 59.90 |
| Refrigerante | Bebida | 7.90 |
| Detergente | Limpeza | 2.90 |

**Consulta:**

```sql
SELECT Categoria,
       AVG(Preco) AS Media,
       SUM(Preco) AS Soma,
       MAX(Preco) AS Maior,
       MIN(Preco) AS Menor
FROM Produto
GROUP BY Categoria;
```

**Resultado:**

| Categoria | Media | Soma | Maior | Menor |
| --- | --- | --- | --- | --- |
| Bebida | 33.90 | 67.80 | 59.90 | 7.90 |
| Limpeza | 6.40 | 12.80 | 9.90 | 2.90 |
| Mercearia | 9.595 | 19.19 | 10.20 | 8.99 |

> ### 2.1.1 Regra do `GROUP BY`
> - **Toda coluna que aparece no `SELECT` sem função de agregação deve aparecer no `GROUP BY`.**
> - Caso contrário, o SGBD pode gerar erro ou comportamento imprevisível.

## 3. `HAVING` – Filtrando grupos

- `HAVING` é usado para filtrar **grupos** (resultados do `GROUP BY`), assim como `WHERE` filtra linhas.
- É aplicado **após** o agrupamento.

### 3.1 Exemplo com `HAVING`

```sql
SELECT Categoria, AVG(Preco) AS Media
FROM Produto
GROUP BY Categoria
HAVING AVG(Preco) < 10;
```

**Resultado (filtra apenas categorias com média < 10):**

| Categoria | Media |
| --- | --- |
| Limpeza | 6.40 |
| Mercearia | 9.595 |

### 3.2 Ordem de execução (importante!)

```
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
```

| CLÁUSULA | QUANDO ATUA |
| --- | --- |
| `WHERE` | Filtra **linhas** antes do agrupamento. |
| `GROUP BY` | Agrupa as linhas. |
| `HAVING` | Filtra os **grupos** após o agrupamento. |
| `SELECT` | Projeta as colunas finais. |
| `ORDER BY` | Ordena o resultado final. |

> ### 3.2.1 Diferença entre `WHERE` e `HAVING`
> - `WHERE` filtra linhas **antes** do agrupamento.
> - `HAVING` filtra grupos **depois** do agrupamento.
> - Não se pode usar funções de agregação no `WHERE` (ex.: `WHERE AVG(Preco) > 10`), mas pode-se no `HAVING`.

## 4. `INSERT` – Inserindo Dados

### 4.1 Sintaxe 1 – sem especificar colunas

```sql
INSERT INTO Produto
VALUES (1, 'TV LCD 50', 3500, 'ELETRONICOS');
```

- A ordem dos valores **deve corresponder** à ordem das colunas na tabela.
- **Não recomendado:** se a estrutura da tabela mudar, o comando pode falhar.

### 4.2 Sintaxe 2 – especificando colunas (recomendado)

```sql
INSERT INTO Produto (ID, Categoria, Nome)
VALUES (1, 'ELETRONICOS', 'TV LCD 50');
```

- A ordem dos valores **não precisa** seguir a ordem da tabela.
- Colunas não listadas recebem valor `NULL` (ou valor padrão, se definido).

### 4.3 Sintaxe 3 – com `SELECT` (inserção a partir de outra tabela)

```sql
INSERT INTO Produto
SELECT * FROM Produto2
WHERE Categoria = 'HIGIENE';
```

- Insere na tabela `Produto` os resultados da consulta sobre `Produto2`.
- As estruturas das tabelas **devem ser compatíveis** (mesmo número e tipos de colunas).

## 5. `DELETE` – Excluindo Dados

### 5.1 Sintaxe básica

```sql
DELETE FROM nome_tabela
WHERE condição;
```

### 5.2 Exemplos

```sql
-- Exclui produtos com preço nulo
DELETE FROM Produto WHERE Preco IS NULL;

-- Exclui TODOS os produtos (cuidado!)
DELETE FROM Produto;
```

> ### 5.2.1 Atenção (pegadinha)
> - `DELETE` sem `WHERE` exclui **todos os registros** da tabela.
> - Use sempre com **muita cautela**.

## 6. `UPDATE` – Atualizando Dados

### 6.1 Sintaxe básica

```sql
UPDATE nome_tabela
SET coluna1 = valor1, coluna2 = valor2
WHERE condição;
```

### 6.2 Exemplos

```sql
-- Aumenta o preço de todos os produtos em 10%
UPDATE Produto SET Preco = Preco * 1.1;

-- Aumenta o preço apenas dos produtos de limpeza em 10%
UPDATE Produto
SET Preco = Preco * 1.1
WHERE Categoria = 'LIMPEZA';
```

> ### 6.2.1 Atenção (pegadinha)
> - `UPDATE` sem `WHERE` afeta **todos os registros** da tabela.
> - Sempre verifique a condição antes de executar.

## 7. Tabela Comparativa – `INSERT`, `DELETE`, `UPDATE`

| COMANDO | FUNÇÃO | SEM `WHERE` | COM `WHERE` |
| --- | --- | --- | --- |
| `INSERT` | Inserir registros | N/A | N/A |
| `DELETE` | Excluir registros | Exclui **todos** os registros | Exclui apenas os que atendem à condição |
| `UPDATE` | Atualizar registros | Atualiza **todos** os registros | Atualiza apenas os que atendem à condição |

## 8. Síntese para Revisão Rápida

| COMANDO / CLÁUSULA | FUNÇÃO |
| --- | --- |
| `COUNT`, `AVG`, `SUM`, `MAX`, `MIN` | Funções de agregação. |
| `GROUP BY` | Agrupa registros (usado com funções de agregação). |
| `HAVING` | Filtra grupos (resultados do `GROUP BY`). |
| `INSERT INTO` | Insere novos registros. |
| `DELETE FROM` | Exclui registros. |
| `UPDATE ... SET` | Atualiza registros. |
| `WHERE` | Filtra linhas **antes** do agrupamento. |
| `ORDER BY` | Ordena o resultado final. |
