# Anotações – Banco de Dados – Linguagem SQL IV (Joins)

## 1. Conceito de JOIN

- **JOIN** é usado para combinar dados de **duas ou mais tabelas** com base em uma condição de relacionamento (geralmente chave estrangeira).
- As tabelas são combinadas por meio de **colunas em comum** (ex.: chave primária/estrangeira).

## 2. Inner Join

### 2.1 Definição

- Retorna apenas os registros que têm **correspondência em ambas as tabelas**.
- Equivalente à **interseção** entre os conjuntos.

### 2.2 Sintaxe

```sql
SELECT p.nome_prod AS Produto, c.nome_categoria AS Categoria
FROM Produto AS p
INNER JOIN Categoria AS c ON (p.id_categoria = c.id_categoria);
```

### 2.3 Exemplo

**Produto:**

| Nome_Prod | ID_Categoria | Preco_Prod |
| --- | --- | --- |
| Café | 10 | 8.99 |
| Açúcar | 10 | 10.20 |
| Sabão em Pó | 20 | 9.90 |
| Vinho | 30 | 59.99 |
| Refrigerante | NULL | 7.90 |

**Categoria:**

| ID_Categoria | Nome_Categoria |
| --- | --- |
| 10 | Mercearia |
| 20 | Limpeza |
| 30 | Bebida |
| 40 | Açougue |

**Resultado do INNER JOIN:**

| Produto | Categoria |
| --- | --- |
| Café | Mercearia |
| Açúcar | Mercearia |
| Sabão em Pó | Limpeza |
| Vinho | Bebida |

> ### 2.3.1 Observação
> - O **Refrigerante** não aparece porque não possui categoria cadastrada (`ID_Categoria = NULL`).
> - A categoria **Açougue** não aparece porque não há produtos associados a ela.

## 3. Left Outer Join (LEFT JOIN)

### 3.1 Definição

- Retorna **todos os registros da tabela à esquerda** (primeira tabela citada) **+** os registros correspondentes da tabela à direita.
- Se não houver correspondência, os campos da tabela à direita ficam com `NULL`.

### 3.2 Sintaxe

```sql
SELECT p.nome_prod AS Produto, c.nome_categoria AS Categoria
FROM Produto AS p
LEFT OUTER JOIN Categoria AS c ON (p.id_categoria = c.id_categoria);
```

### 3.3 Resultado

| Produto | Categoria |
| --- | --- |
| Café | Mercearia |
| Açúcar | Mercearia |
| Sabão em Pó | Limpeza |
| Vinho | Bebida |
| Refrigerante | `NULL` |

> ### 3.3.1 Observação
> - **Refrigerante** aparece, mesmo sem categoria (`NULL`).
> - A categoria **Açougue** (que está na tabela à direita) **não aparece**, porque o LEFT JOIN prioriza a tabela da esquerda.

## 4. Right Outer Join (RIGHT JOIN)

### 4.1 Definição

- Retorna **todos os registros da tabela à direita** (segunda tabela citada) **+** os registros correspondentes da tabela à esquerda.
- Se não houver correspondência, os campos da tabela à esquerda ficam com `NULL`.

### 4.2 Sintaxe

```sql
SELECT p.nome_prod AS Produto, c.nome_categoria AS Categoria
FROM Produto AS p
RIGHT OUTER JOIN Categoria AS c ON (p.id_categoria = c.id_categoria);
```

### 4.3 Resultado

| Produto | Categoria |
| --- | --- |
| Café | Mercearia |
| Açúcar | Mercearia |
| Sabão em Pó | Limpeza |
| Vinho | Bebida |
| `NULL` | Açougue |

> ### 4.3.1 Observação
> - **Açougue** aparece, mesmo sem produtos associados (`NULL` no campo Produto).
> - **Refrigerante** não aparece, porque o RIGHT JOIN prioriza a tabela da direita (Categoria), e Refrigerante está apenas na tabela da esquerda (Produto).

## 5. Full Outer Join (FULL JOIN)

### 5.1 Definição

- Retorna **todos os registros de ambas as tabelas** (união).
- Se não houver correspondência, os campos da tabela sem correspondência ficam com `NULL`.

### 5.2 Sintaxe

```sql
SELECT p.nome_prod AS Produto, c.nome_categoria AS Categoria
FROM Produto AS p
FULL OUTER JOIN Categoria AS c ON (p.id_categoria = c.id_categoria);
```

### 5.3 Resultado

| Produto | Categoria |
| --- | --- |
| Café | Mercearia |
| Açúcar | Mercearia |
| Sabão em Pó | Limpeza |
| Vinho | Bebida |
| `NULL` | Açougue |
| Refrigerante | `NULL` |

> ### 5.3.1 Observação
> - O FULL JOIN é a **combinação** do LEFT JOIN e do RIGHT JOIN.
> - Todos os registros de ambas as tabelas aparecem.

## 6. Auto-Relacionamento (Self JOIN)

### 6.1 Definição

- É um **INNER JOIN** onde a tabela é **relacionada com ela mesma**.
- Útil quando há referência dentro da própria tabela (ex.: empregado e chefe, onde o chefe também é um empregado).

### 6.2 Exemplo

**Tabela EMPREGADO:**

| ID_Emp | Nome | Setor | ID_Chefe |
| --- | --- | --- | --- |
| 10 | Jose | TI | NULL |
| 20 | Maria | TI | 10 |
| 30 | Marta | TI | 10 |
| 40 | Antonio | TI | 20 |
| 50 | Manoel | TI | 30 |
| 60 | Carla | JUR | NULL |
| 70 | Renato | JUR | 60 |
| 80 | Julia | JUR | 60 |
| 90 | Jonas | JUR | 70 |
| 100 | Paulo | JUR | 80 |

### 6.3 Consulta (INNER JOIN com autorrelacionamento)

```sql
SELECT E.nome AS Empregado, C.nome AS Chefe
FROM Empregado AS E
INNER JOIN Empregado AS C ON (E.id_chefe = C.id_emp);
```

### 6.4 Resultado

| Empregado | Chefe |
| --- | --- |
| Maria | Jose |
| Marta | Jose |
| Antonio | Maria |
| Manoel | Marta |
| Renato | Carla |
| Julia | Carla |
| Jonas | Renato |
| Paulo | Julia |

> ### 6.4.1 Observação
> - Empregados sem chefe (`ID_Chefe = NULL`) não aparecem (Jose e Carla).
> - Cada empregado é relacionado ao seu chefe correspondente.

## 7. Síntese Comparativa – Tipos de JOIN

| TIPO DE JOIN | TABELA ESQUERDA | TABELA DIREITA | `NULL` PODE APARECER | EQUIVALÊNCIA |
| --- | --- | --- | --- | --- |
| **INNER JOIN** | Apenas correspondências | Apenas correspondências | Não | Interseção |
| **LEFT JOIN** | Todos os registros | Apenas correspondências | Sim (lado direito) | Tabela esquerda + interseção |
| **RIGHT JOIN** | Apenas correspondências | Todos os registros | Sim (lado esquerdo) | Tabela direita + interseção |
| **FULL JOIN** | Todos os registros | Todos os registros | Sim (ambos os lados) | União |
| **Self JOIN** | A própria tabela (como se fossem duas) | A própria tabela (como se fossem duas) | Depende do tipo de JOIN usado | Relacionamento dentro da mesma tabela |

> ### 7.1 Ordem de prioridade dos JOINs
> - **Esquerda (LEFT):** tabela antes do `LEFT JOIN`.
> - **Direita (RIGHT):** tabela depois do `RIGHT JOIN`.
> - **FULL:** ambos os lados são priorizados.

## 8. Dica para Provas

| SITUAÇÃO | TIPO DE JOIN RECOMENDADO |
| --- | --- |
| Quer apenas registros que existem em ambas as tabelas | `INNER JOIN` |
| Quer todos os registros da tabela principal mesmo sem correspondência | `LEFT JOIN` (tabela principal à esquerda) |
| Quer todos os registros da tabela secundária mesmo sem correspondência | `RIGHT JOIN` (tabela secundária à direita) |
| Quer todos os registros de ambas as tabelas | `FULL JOIN` |
| Relacionamento dentro da mesma tabela | `SELF JOIN` (com alias) |
